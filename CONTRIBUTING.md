# Contributing to PHP Ruleset Formatter

Thank you for your interest in contributing! Your help is greatly appreciated. Please follow these guidelines to make the process smooth for everyone.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Clone your fork** and set up the project:
   ```sh
   npm install
   ```
3. **Create clear, descriptive commits** and keep your branch up to date with `main`.
4. **Test your changes** before submitting a pull request.
5. **Submit a pull request** with a clear description of your changes and the problem they solve.

## Reporting Issues

- Use the [GitHub Issues](../../issues) page to report bugs or request features.
- Please provide as much detail as possible, including steps to reproduce and screenshots if relevant.

## Coding Standards

- Follow the existing code style (TypeScript, consistent formatting).
- Use meaningful variable and function names.
- Add comments where necessary for clarity.
- Run `npm run compile` to ensure your code builds without errors.

## Pull Request Checklist

- [ ] Code builds and passes all checks
- [ ] New/updated tests (if applicable)
- [ ] Documentation updated (if needed)
- [ ] Linked to relevant issue (if applicable)

## Contact

For questions, open an issue or contact the maintainer via GitHub.

Thank you for helping improve PHP Ruleset Formatter!
